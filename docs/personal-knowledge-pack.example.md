# Personal Knowledge Pack Example

This file demonstrates the local personal knowledge-pack format used by EchoGuide.
Do not fill it with private data. Keep the real file local, for example at
`.echoguide/knowledge.local.md`.

## Profile Summary

- Current positioning: AI integration engineer with backend, mobile and agentic
  development background.
- Communication style: concise, concrete, practical.

## Current Target Tracks

- AI Integration / LLM
- AI Harness Engineer

## Projects

### Example Project Name

- Context: short public-safe description of the product or internal tool.
- Role: what you personally owned.
- Problem: what was unclear, slow or expensive before the project.
- Approach: architecture, API, automation or workflow choices.
- Result: measurable or observable outcome. Use `clarify` if the number is not
  confirmed.
- Technologies: TypeScript, Python, OpenAI API, PostHog, Docker.

## Roles And Responsibilities

- Reduced manual review time by `clarify`.

## STAR Stories

### Handling Unclear Requirements

- Situation: requirements were incomplete.
- Task: make the next implementation step safe and testable.
- Action: documented assumptions, built a small validation path, added checks.
- Result: team could review real output instead of discussing abstractions.

## Phrases To Avoid

- Do not claim people management unless the role explicitly required it.
- Do not invent exact revenue, traffic or team-size numbers.
- Do not present prototypes as production systems.

## Preferred Answer Style

- Start with a compact direct answer.
- Mention one concrete project fact.
- Offer to go deeper into architecture, trade-offs or metrics.
- Avoid long monologues unless the interviewer asks for detail.
